Tutorial: Estructuras de Datos en Python

Organiza y manipula colecciones de datos con Listas, Tuplas, Diccionarios y Sets

🏗️

¿Qué son las Estructuras de Datos?

¡Bienvenido/a al mundo de la organización de datos! Hasta ahora, has trabajado principalmente con variables individuales que guardan un solo valor (un número, un texto, un booleano).

Pero, ¿qué pasa cuando necesitas agrupar varios datos relacionados? Aquí es donde entran las estructuras de datos. Son formas especializadas de organizar, almacenar y gestionar colecciones de datos para que puedan ser accedidos y modificados eficientemente.

Python ofrece varias estructuras de datos incorporadas. En este tutorial, nos enfocaremos en cuatro de las más importantes:

  • Listas (Lists): Colecciones ordenadas y modificables.
  • Tuplas (Tuples): Colecciones ordenadas e inmutables (no modificables después de su creación).
  • Diccionarios (Dictionaries): Colecciones desordenadas (en Python < 3.7, ordenadas por inserción en >= 3.7) de pares clave-valor.
  • Conjuntos (Sets): Colecciones desordenadas de elementos únicos.
Elegir la estructura de datos correcta puede hacer tu código más eficiente, legible y fácil de mantener.
📋

Listas: List

Las listas son una de las estructuras de datos más versátiles y comúnmente usadas en Python.

Características principales de las listas:

  • Ordenadas: Los elementos se almacenan en un orden específico y ese orden se mantiene.
  • Modificables (Mutables): Puedes cambiar, añadir o eliminar elementos después de crear la lista.
  • Permiten duplicados: Una lista puede contener el mismo valor múltiples veces.
  • Heterogéneas: Pueden contener elementos de diferentes tipos de datos (aunque es común que contengan elementos del mismo tipo).

Se crean usando corchetes [ ] y separando los elementos con comas.

# Lista vacía
mi_lista_vacia = []

# Lista de números
numeros = [1, 2, 3, 4, 5]

# Lista de strings
frutas = ["manzana", "banana", "cereza"]

# Lista mixta
mixta = ["hola", 3.14, True, 42]

print(frutas)

Salida esperada:

['manzana', 'banana', 'cereza']
🔧

Listas: Acceso y Modificación

Puedes acceder a los elementos de una lista usando su índice (posición). El primer elemento tiene índice 0, el segundo 1, y así sucesivamente.

colores = ["rojo", "verde", "azul", "amarillo"]

print(colores[0])  # Accede al primer elemento
print(colores[2])  # Accede al tercer elemento

# También puedes usar índices negativos para contar desde el final
print(colores[-1]) # Accede al último elemento

Salida esperada:

rojo azul amarillo

Como las listas son modificables, puedes cambiar un elemento:

colores[1] = "morado" # Cambia "verde" por "morado"
print(colores)

Salida esperada:

['rojo', 'morado', 'azul', 'amarillo']
Si intentas acceder a un índice que no existe (ej. colores[10] en una lista de 4 elementos), obtendrás un error IndexError.
🛠️

Listas: Métodos Comunes

Las listas tienen muchos métodos útiles (funciones asociadas a ellas):

  • append(elemento): Añade un elemento al final de la lista.
  • insert(indice, elemento): Inserta un elemento en una posición específica.
  • remove(elemento): Elimina la primera aparición del elemento especificado.
  • pop(indice_opcional): Elimina y devuelve el elemento en el índice dado (o el último si no se especifica índice).
  • sort(): Ordena la lista (modifica la original).
  • len(lista): (Función, no método) Devuelve el número de elementos en la lista.
numeros = [3, 1, 4]
numeros.append(2)          # numeros ahora es [3, 1, 4, 2]
print(f"Después de append: {numeros}")

numeros.insert(1, 5)       # numeros ahora es [3, 5, 1, 4, 2]
print(f"Después de insert: {numeros}")

numeros.sort()             # numeros ahora es [1, 2, 3, 4, 5]
print(f"Después de sort: {numeros}")

print(f"Longitud de la lista: {len(numeros)}")

Salida esperada:

Después de append: [3, 1, 4, 2] Después de insert: [3, 5, 1, 4, 2] Después de sort: [1, 2, 3, 4, 5] Longitud de la lista: 5
🔒

Tuplas: Tuple

Las tuplas son similares a las listas, pero con una diferencia clave: ¡son inmutables! Esto significa que una vez que creas una tupla, no puedes cambiar su contenido (ni añadir, ni eliminar, ni modificar elementos).

Características principales de las tuplas:

  • Ordenadas: Los elementos se almacenan en un orden específico.
  • Inmutables: No se pueden modificar después de su creación.
  • Permiten duplicados.
  • Heterogéneas.

Se crean usando paréntesis ( ) y separando los elementos con comas. Los paréntesis son opcionales en muchos casos si la coma está presente, pero es buena práctica usarlos para mayor claridad.

# Tupla de coordenadas
punto = (10, 20)

# Tupla de colores primarios (no se espera que cambien)
rgb = ("rojo", "verde", "azul")

# Tupla con un solo elemento (¡necesita una coma al final!)
mi_tupla_unica = ("valor",) 

print(punto)
print(rgb[1]) # Se accede igual que a las listas

Salida esperada:

(10, 20) verde
Intentar modificar una tupla resultará en un error TypeError. 
rgb[0] = "magenta"  # Esto daría error
¿Cuándo usar tuplas? Cuando tienes una colección de elementos que no debe cambiar, como coordenadas, constantes, o registros de datos fijos. También son ligeramente más eficientes que las listas en algunos contextos debido a su inmutabilidad.
🔑

Diccionarios: dict

Los diccionarios son colecciones que almacenan datos en pares clave-valor. En lugar de acceder a los elementos por un índice numérico, se accede mediante una clave única.

Características principales de los diccionarios:

  • Pares Clave-Valor: Cada elemento es una asociación de una clave con su valor.
  • Claves Únicas: Las claves dentro de un diccionario deben ser únicas.
  • Modificables (Mutables): Puedes añadir, eliminar o modificar pares clave-valor.
  • Ordenados (Python 3.7+): Desde Python 3.7, los diccionarios recuerdan el orden de inserción. En versiones anteriores, eran desordenados.
  • Las claves suelen ser strings o números (deben ser de tipo inmutable). Los valores pueden ser de cualquier tipo.

Se crean usando llaves { } con pares clave: valor separados por comas.

# Diccionario vacío
diccionario_vacio = {}

# Información de una persona
persona = {
    "nombre": "Ana",
    "edad": 30,
    "ciudad": "México",
    "profesion": "Ingeniera"
}
print(persona)

Salida esperada (el orden puede variar en Python < 3.7):

{'nombre': 'Ana', 'edad': 30, 'ciudad': 'México', 'profesion': 'Ingeniera'}
🔧

Diccionarios: Acceso y Modificación

Se accede a los valores usando sus claves entre corchetes [ ].

persona = {"nombre": "Carlos", "edad": 25}

print(persona["nombre"])
print(persona["edad"])

Salida esperada:

Carlos 25

Puedes añadir nuevos pares clave-valor o modificar existentes:

persona["ciudad"] = "Guadalajara"  # Añadir nueva clave-valor
persona["edad"] = 26              # Modificar valor existente
print(persona)

Salida esperada:

{'nombre': 'Carlos', 'edad': 26, 'ciudad': 'Guadalajara'}
Si intentas acceder a una clave que no existe usando diccionario[clave_inexistente], obtendrás un error KeyError.
🛠️

Diccionarios: Métodos Comunes

Algunos métodos útiles para diccionarios:

  • get(clave, valor_por_defecto_opcional): Devuelve el valor de la clave. Si la clave no existe, devuelve None o el valor por defecto si se proporciona (evita KeyError).
  • keys(): Devuelve una vista de todas las claves.
  • values(): Devuelve una vista de todos los valores.
  • items(): Devuelve una vista de todos los pares (clave, valor).
  • pop(clave, valor_por_defecto_opcional): Elimina la clave y devuelve su valor.
producto = {"id": 101, "nombre": "Laptop", "precio": 15000}

print(producto.get("nombre"))
print(producto.get("stock", 0)) # "stock" no existe, devuelve 0

print("Claves:", list(producto.keys()))  # Convertimos a lista para verlas bien
print("Valores:", list(producto.values()))

for clave, valor in producto.items():
    print(f"{clave}: {valor}")

Salida esperada (el orden de claves/valores puede variar en Python < 3.7 para .keys() y .values() si no se convierten a lista):

Laptop 0 Claves: ['id', 'nombre', 'precio'] Valores: [101, 'Laptop', 15000] id: 101 nombre: Laptop precio: 15000
🧩

Conjuntos: Set

Los conjuntos (sets) son colecciones desordenadas de elementos únicos. Esto significa que no pueden tener elementos duplicados.

Características principales de los conjuntos:

  • Desordenados: No mantienen un orden específico de los elementos (aunque al imprimirlos pueden parecer ordenados a veces, no debes confiar en ese orden).
  • Elementos Únicos: No permiten duplicados. Si intentas añadir un elemento que ya existe, simplemente no se añade.
  • Modificables (Mutables): Puedes añadir o eliminar elementos.
  • Los elementos de un conjunto deben ser de tipo inmutable (ej. números, strings, tuplas). No puedes tener listas dentro de un conjunto.

Se crean usando llaves { } o la función set().

# Creando un conjunto con llaves
numeros_unicos = {1, 2, 3, 2, 4, 1} # Los duplicados se eliminan automáticamente
print(numeros_unicos)

# Creando un conjunto a partir de una lista (para eliminar duplicados)
mi_lista = ['a', 'b', 'a', 'c', 'b', 'd']
conjunto_de_letras = set(mi_lista)
print(conjunto_de_letras)

# ¡Cuidado! Para crear un conjunto vacío, DEBES usar set()
conjunto_vacio_ok = set()
diccionario_vacio_no_set = {} # Esto crea un diccionario vacío, no un conjunto

Salida esperada (el orden de los elementos puede variar):

{1, 2, 3, 4} {'b', 'c', 'd', 'a'} <-- o cualquier otro orden
➕➖

Conjuntos: Operaciones

Los conjuntos son muy útiles para operaciones matemáticas de conjuntos como unión, intersección y diferencia.

  • add(elemento): Añade un elemento al conjunto.
  • remove(elemento): Elimina un elemento. Si no existe, da KeyError.
  • discard(elemento): Elimina un elemento si está presente. No da error si no existe.
  • union(otro_conjunto) o |: Devuelve un nuevo conjunto con elementos de ambos.
  • intersection(otro_conjunto) o &: Devuelve un nuevo conjunto con elementos comunes a ambos.
  • difference(otro_conjunto) o -: Devuelve un nuevo conjunto con elementos del primer conjunto que no están en el segundo.
set_a = {1, 2, 3, 4}
set_b = {3, 4, 5, 6}

set_a.add(5) # set_a ahora es {1, 2, 3, 4, 5}
print(f"Set A después de add: {set_a}")

union_sets = set_a.union(set_b) # o set_a | set_b
print(f"Unión: {union_sets}")

interseccion_sets = set_a.intersection(set_b) # o set_a & set_b
print(f"Intersección: {interseccion_sets}")

Salida esperada (el orden puede variar):

Set A después de add: {1, 2, 3, 4, 5} Unión: {1, 2, 3, 4, 5, 6} Intersección: {3, 4, 5}
⚖️

Comparación: ¿Cuándo Usar Cuál?

Aquí tienes una tabla resumen para ayudarte a decidir qué estructura usar:

Característica Lista (List) [] Tupla (Tuple) () Diccionario (Dict) {} Conjunto (Set) {} o set()
Ordenada Sí Sí Sí (Python 3.7+) No
Modificable (Mutable) Sí No (Inmutable) Sí Sí (pero sus elementos deben ser inmutables)
Permite Duplicados Sí Sí No para claves (valores sí) No
Acceso Por índice numérico (lista[0]) Por índice numérico (tupla[0]) Por clave (diccionario['clave']) No directamente por índice (se itera o verifica pertenencia)
Uso Común Colecciones generales, secuencias que cambian. Datos que no deben cambiar (coordenadas, constantes). Mapeo de claves a valores, búsquedas rápidas por clave. Verificar pertenencia, eliminar duplicados, operaciones de conjuntos.

Iterando sobre Estructuras

Puedes usar bucles for para recorrer la mayoría de estas estructuras:

Listas y Tuplas:

mi_lista = ['a', 'b', 'c']
for elemento in mi_lista:
    print(elemento)

Diccionarios:

mi_diccionario = {'x': 1, 'y': 2}
# Iterar sobre claves (por defecto)
for clave in mi_diccionario:
    print(f"Clave: {clave}, Valor: {mi_diccionario[clave]}")

# Iterar sobre pares clave-valor
for clave, valor in mi_diccionario.items():
    print(f"{clave} -> {valor}")

Conjuntos:

mi_conjunto = {10, 20, 30}
for numero in mi_conjunto:
    print(numero) # El orden no está garantizado

Estructuras de Datos Anidadas

Puedes tener estructuras de datos dentro de otras estructuras de datos. Esto es muy común y poderoso.

Ejemplo: Una lista de diccionarios.

estudiantes = [
    {"nombre": "Luis", "edad": 20, "cursos": ["Matemáticas", "Física"]},
    {"nombre": "Elena", "edad": 22, "cursos": ["Historia", "Literatura", "Arte"]}
]

# Acceder al primer curso del segundo estudiante
print(estudiantes[1]["cursos"][0])

# Iterar para mostrar nombres y número de cursos
for est in estudiantes:
    print(f"{est['nombre']} está inscrito en {len(est['cursos'])} cursos.")

Salida esperada:

Historia Luis está inscrito en 2 cursos. Elena está inscrita en 3 cursos.
Comprender cómo acceder a elementos en estructuras anidadas es clave. Ve "capa por capa" con los corchetes.
