¿Qué son las Funciones?

¡Llegamos a un concepto fundamental en programación: las funciones!

Una función es un bloque de código organizado y reutilizable que realiza una tarea específica. Piensa en ellas como mini-programas dentro de tu programa principal.

Beneficios de usar funciones:

  • Reutilización de código: Escribes el código una vez y lo puedes llamar (usar) muchas veces.
  • Modularidad: Dividen un programa grande en partes más pequeñas y manejables.
  • Legibilidad: Hacen que el código sea más fácil de entender, ya que cada función tiene un propósito claro.
  • Abstracción: Ocultan los detalles complejos de una tarea detrás de un nombre simple.
Python tiene muchas funciones incorporadas (built-in) como print(), len(), input(), que ya has estado usando. ¡Ahora aprenderás a crear las tuyas!
✍️

Definir y Llamar una Función

Para crear (definir) una función en Python, usamos la palabra clave def, seguida del nombre de la función, paréntesis (), y dos puntos :. El código dentro de la función debe estar indentado.

Sintaxis básica para definir una función:

def nombre_de_la_funcion():
    # Código que realiza la función
    # (Debe estar indentado)
    print("¡Esta función ha sido llamada!")

Para usar (llamar o invocar) la función, simplemente escribes su nombre seguido de paréntesis:

nombre_de_la_funcion() # Así se llama a la función

Ejemplo completo:

# Definición de la función
def saludar():
    print("¡Hola desde la función saludar!")

# Llamada a la función
saludar()
saludar() # Podemos llamarla cuantas veces queramos

Salida esperada:

¡Hola desde la función saludar! ¡Hola desde la función saludar!
📥

Pasando Información: Parámetros y Argumentos

Las funciones pueden recibir información para trabajar con ella. Esta información se pasa a través de parámetros (variables definidas en la declaración de la función) y argumentos (los valores reales que se envían al llamar la función).

def saludar_a(nombre):  # 'nombre' es un parámetro
    print(f"¡Hola, {nombre}! ¿Cómo estás?")

saludar_a("Ana")         # "Ana" es un argumento
saludar_a("Dr. Muñoz")   # "Dr. Muñoz" es otro argumento

Salida esperada:

¡Hola, Ana! ¿Cómo estás? ¡Hola, Dr. Muñoz! ¿Cómo estás?

Una función puede tener múltiples parámetros:

def sumar(a, b):
    resultado = a + b
    print(f"La suma de {a} y {b} es {resultado}")

sumar(5, 3)
sumar(100, 200)

Salida esperada:

La suma de 5 y 3 es 8 La suma de 100 y 200 es 300
📤

Devolviendo Valores: La Sentencia return

Hasta ahora, nuestras funciones solo han impreso cosas. Pero a menudo, queremos que una función calcule un valor y nos lo devuelva para que podamos usarlo en otra parte de nuestro código.

Para esto, usamos la sentencia return.

def multiplicar(x, y):
    producto = x * y
    return producto  # Devuelve el valor de 'producto'

resultado1 = multiplicar(4, 5) # El valor devuelto se asigna a resultado1
resultado2 = multiplicar(10, 3)

print(f"Resultado 1: {resultado1}")
print(f"Resultado 2: {resultado2}")
print(f"Suma de resultados: {resultado1 + resultado2}")

Salida esperada:

Resultado 1: 20 Resultado 2: 30 Suma de resultados: 50
Una función puede tener múltiples sentencias return (por ejemplo, dentro de un if/else). Sin embargo, tan pronto como se ejecuta una sentencia return, la función termina y devuelve ese valor. Cualquier código después de un return en la misma ruta de ejecución no se ejecutará.

Si una función no tiene una sentencia return explícita, o tiene un return sin un valor, implícitamente devuelve None.

⚙️

Parámetros por Defecto y Argumentos Nombrados

Parámetros por Defecto: Puedes asignar un valor por defecto a un parámetro en la definición de la función. Si al llamar la función no se proporciona un argumento para ese parámetro, se usará el valor por defecto.

def potencia(base, exponente=2): # 'exponente' tiene valor por defecto 2
    return base ** exponente

print(potencia(3))        # Usa exponente=2 por defecto (3^2)
print(potencia(3, 3))      # Proporcionamos exponente=3 (3^3)

Salida esperada:

9 27
Los parámetros con valores por defecto deben ir después de los parámetros sin valores por defecto en la definición de la función.

Argumentos Nombrados (Keyword Arguments): Al llamar a una función, puedes especificar a qué parámetro corresponde cada argumento usando el nombre del parámetro.

def describir_mascota(tipo_animal, nombre_mascota):
    print(f"Tengo un {tipo_animal}.")
    print(f"Mi {tipo_animal} se llama {nombre_mascota}.")

# Llamada posicional (el orden importa)
describir_mascota("perro", "Firulais")

# Llamada con argumentos nombrados (el orden NO importa)
describir_mascota(nombre_mascota="Mishi", tipo_animal="gato")

Esto mejora la legibilidad, especialmente con funciones que tienen muchos parámetros.

🌍📍

Ámbito de las Variables (Scope)

El ámbito (o scope) de una variable se refiere a la parte del programa donde esa variable es accesible.

  • Variables Locales: Se definen dentro de una función. Solo son accesibles desde dentro de esa función.
  • Variables Globales: Se definen fuera de todas las funciones. Son accesibles desde cualquier parte del código (dentro y fuera de las funciones).
variable_global = "Soy global"

def mi_funcion():
    variable_local = "Soy local"
    print(variable_local)       # Acceso a variable local (OK)
    print(variable_global)      # Acceso a variable global (OK)

mi_funcion()

print(variable_global)        # Acceso a variable global (OK)
# print(variable_local)      # Esto daría un NameError, 'variable_local' no está definida aquí

Salida esperada (sin la línea comentada):

Soy local Soy global Soy global
Si una variable local tiene el mismo nombre que una global, dentro de la función, la local "oculta" a la global. Para modificar una variable global desde dentro de una función, necesitarías usar la palabra clave global (aunque su uso excesivo se considera mala práctica).
📜

Documentando Funciones: Docstrings

Es una buena práctica documentar tus funciones para explicar qué hacen, qué parámetros reciben y qué devuelven. En Python, esto se hace con docstrings (cadenas de documentación).

Un docstring es una cadena de texto (generalmente multilínea, encerrada en triples comillas """ """ o ''' ''') que se coloca como la primera línea dentro de la definición de una función.

def calcular_area_rectangulo(base, altura):
    """
    Calcula el área de un rectángulo.

    Parámetros:
    base (float): La longitud de la base del rectángulo.
    altura (float): La altura del rectángulo.

    Devuelve:
    float: El área calculada del rectángulo.
    """
    if base < 0 or altura < 0:
        return "Las dimensiones deben ser positivas."
    return base * altura

# Puedes acceder al docstring de una función usando el atributo __doc__
print(calcular_area_rectangulo.__doc__)

# O usando la función help()
# help(calcular_area_rectangulo) # Descomenta para probar en un entorno Python

Escribir buenos docstrings es crucial para que otros (¡y tu yo del futuro!) puedan entender y usar tus funciones correctamente.

✨

Argumentos Variables: *args

A veces, quieres que una función pueda aceptar un número variable de argumentos posicionales. Python te permite hacer esto usando la sintaxis *args.

Dentro de la función, args será una tupla que contendrá todos los argumentos posicionales extra que se pasaron.

def sumar_todos(*numeros): # 'numeros' será una tupla
    """Suma todos los números pasados como argumentos."""
    total = 0
    for num in numeros:
        total += num
    return total

print(sumar_todos(1, 2, 3))
print(sumar_todos(10, 20, 30, 40, 50))
print(sumar_todos()) # También funciona sin argumentos

Salida esperada:

6 150 0
El nombre args es una convención, podrías usar *mi_tupla_de_argumentos, pero *args es universalmente entendido. Puedes tener parámetros normales antes de *args.
🔑✨

Argumentos de Palabra Clave Variables: **kwargs

De manera similar a *args, Python permite que una función acepte un número variable de argumentos nombrados (de palabra clave) usando la sintaxis **kwargs.

Dentro de la función, kwargs será un diccionario que contendrá todos los argumentos nombrados extra que se pasaron.

def mostrar_info(**datos_persona): # 'datos_persona' será un diccionario
    """Muestra la información pasada como argumentos nombrados."""
    print("Información recibida:")
    for clave, valor in datos_persona.items():
        print(f"  - {clave}: {valor}")

mostrar_info(nombre="Luis", edad=28, ciudad="Monterrey")
mostrar_info(producto="Laptop", precio=25000, en_stock=True)

Salida esperada:

Información recibida: - nombre: Luis - edad: 28 - ciudad: Monterrey Información recibida: - producto: Laptop - precio: 25000 - en_stock: True
El nombre kwargs (keyword arguments) es una convención. Puedes tener parámetros normales, luego *args, y luego **kwargs, en ese orden: def func(param1, param2, *args, **kwargs):.
🥷

Funciones Lambda (Anónimas)

Python permite crear pequeñas funciones anónimas (sin nombre) usando la palabra clave lambda. Son útiles para funciones simples que se usan una sola vez o como argumentos para funciones de orden superior (como map, filter).

Sintaxis: lambda argumentos: expresion

  • Una función lambda puede tener cualquier número de argumentos, pero solo puede tener una expresión.
  • La expresión se evalúa y se devuelve.
  • No pueden contener múltiples sentencias ni docstrings complejos.
# Función normal para sumar 5
def sumar_cinco_normal(x):
    return x + 5

# Equivalente con lambda
sumar_cinco_lambda = lambda x: x + 5

print(sumar_cinco_normal(10))
print(sumar_cinco_lambda(10))

# Lambda con múltiples argumentos
multiplicar = lambda a, b: a * b
print(multiplicar(7, 6))

Salida esperada:

15 15 42

Se usan a menudo con funciones como map() (aplica una función a todos los ítems de un iterable) o filter() (filtra ítems de un iterable).

🎁

Introducción a Decoradores

Los decoradores son una forma de metaprogramación en Python que permite modificar o mejorar funciones o métodos de una manera limpia y legible. Un decorador es, en esencia, una función que toma otra función como argumento (la función decorada), le añade alguna funcionalidad y devuelve otra función (generalmente una versión modificada de la original o una nueva que la envuelve).

La sintaxis usa el símbolo @ seguido del nombre del decorador, justo antes de la definición de la función que se quiere decorar.

Ejemplo conceptual (muy simplificado):

def mi_decorador(funcion_a_decorar):
    def envoltura(*args, **kwargs):
        print("Algo antes de llamar a la función.")
        resultado = funcion_a_decorar(*args, **kwargs)
        print("Algo después de llamar a la función.")
        return resultado
    return envoltura

@mi_decorador
def saluda(nombre):
    print(f"¡Hola, {nombre}!")

saluda("Mundo")

Salida esperada:

Algo antes de llamar a la función. ¡Hola, Mundo! Algo después de llamar a la función.
Los decoradores son un tema avanzado, pero muy potente. Se usan para logging, control de acceso, instrumentación, y mucho más, sin modificar directamente el código de la función original.
