Entrada y Salida de Datos (I/O)

Una parte fundamental del análisis de datos es poder cargar datos desde diversas fuentes y guardar los resultados de tus análisis.

Pandas ofrece un conjunto robusto de funciones para leer y escribir datos en una amplia variedad de formatos de archivo. Los más comunes son:

  • CSV (Comma-Separated Values): Archivos de texto plano donde los valores están separados por comas (u otro delimitador). Muy común para datos tabulares.
  • Excel (.xls, .xlsx): Hojas de cálculo de Microsoft Excel.
  • JSON (JavaScript Object Notation): Un formato ligero de intercambio de datos, fácil de leer por humanos y máquinas.
  • SQL: Directamente desde y hacia bases de datos relacionales.
  • Y muchos otros como HTML, HDF5, Parquet, etc.

En este tutorial, nos enfocaremos en los más utilizados: CSV y Excel, con una breve mención a JSON.

La capacidad de Pandas para manejar diferentes formatos de archivo lo convierte en una herramienta central en el flujo de trabajo de análisis de datos.
📄

Leyendo Archivos CSV

El formato CSV es uno de los más populares para almacenar datos tabulares. Pandas facilita enormemente la lectura de estos archivos con la función pd.read_csv().

Sintaxis básica: pd.read_csv('ruta/al/archivo.csv')

Esta función tiene muchos parámetros útiles para personalizar la lectura. Algunos comunes son:

  • sep (o delimiter): El carácter que separa los valores (por defecto es ','). Usa '\t' para archivos TSV (tab-separated).
  • header: El número de fila a usar como nombres de columna (por defecto 0, la primera fila). Si no hay encabezado, usa header=None.
  • index_col: Especifica qué columna (o columnas) usar como índice de filas del DataFrame.
  • usecols: Una lista de nombres de columna o índices de columna a leer (para cargar solo un subconjunto).
  • dtype: Un diccionario para especificar el tipo de dato de columnas específicas (ej. {'ColumnaA': str, 'ColumnaB': int}).
  • skiprows: Número de filas a omitir al principio del archivo.
  • nrows: Número de filas a leer del archivo (útil para archivos grandes).

Ejemplo (asumiendo un archivo datos_ejemplo.csv):

Supongamos que datos_ejemplo.csv contiene:

Nombre,Edad,Ciudad Ana,28,Mexico Luis,34,Bogota Sofia,22,Lima
import pandas as pd

# Asumiendo que 'datos_ejemplo.csv' está en el mismo directorio
# df = pd.read_csv('datos_ejemplo.csv')

# Para este ejemplo, crearemos el DataFrame directamente
data_simulada = {
    'Nombre': ['Ana', 'Luis', 'Sofia'],
    'Edad': [28, 34, 22],
    'Ciudad': ['Mexico', 'Bogota', 'Lima']
}
df = pd.DataFrame(data_simulada)
print("DataFrame cargado (simulado):\n", df)

# Ejemplo con parámetros:
# df_personalizado = pd.read_csv('otro_archivo.csv', sep=';', header=None, names=['colA', 'colB'])

Salida esperada (con datos simulados):

DataFrame cargado (simulado): Nombre Edad Ciudad 0 Ana 28 Mexico 1 Luis 34 Bogota 2 Sofia 22 Lima
Siempre revisa las primeras filas (.head()) y la información (.info()) de un DataFrame después de cargarlo para asegurarte de que se leyó correctamente.

Leyendo Archivos Excel y JSON

Leyendo Archivos Excel (.xls, .xlsx):

Pandas usa la función pd.read_excel(). Necesitarás tener instalada una librería adicional como openpyxl (para .xlsx) o xlrd (para .xls más antiguos).

pip install openpyxl

Sintaxis básica: pd.read_excel('ruta/al/archivo.xlsx', sheet_name='Hoja1')

  • sheet_name: El nombre o índice de la hoja a leer (por defecto 0, la primera hoja). Puedes pasar una lista de nombres/índices para leer múltiples hojas en un diccionario de DataFrames.
  • Muchos parámetros son similares a read_csv() (header, index_col, usecols, etc.).
import pandas as pd

Leyendo Archivos JSON (JavaScript Object Notation):

Pandas usa pd.read_json(). JSON puede tener diferentes estructuras, por lo que a veces necesitas especificar cómo interpretarlo (ej. con el parámetro orient).

Sintaxis básica: pd.read_json('ruta/al/archivo.json')

Supongamos que datos.json tiene el siguiente contenido (orientación 'records'):

[{"Nombre":"Ana","Edad":28},{"Nombre":"Luis","Edad":34}]
# df_json = pd.read_json('datos.json', orient='records')
# print(df_json)
La lectura de archivos JSON puede ser más compleja dependiendo de la estructura del JSON. El parámetro orient (ej. 'records', 'columns', 'index', 'split', 'table') es crucial.

Escribiendo DataFrames a Archivos

Después de procesar y analizar tus datos, querrás guardar los resultados. Pandas ofrece métodos para escribir DataFrames a varios formatos.

Escribir a CSV: df.to_csv()

Sintaxis: df.to_csv('nombre_archivo_salida.csv', index=True, header=True, sep=',')

  • index=False: Para no escribir el índice del DataFrame en el archivo. Común si el índice no es significativo.
  • header=True: Para incluir los nombres de columna (por defecto).
  • sep=',': Delimitador (por defecto coma).
import pandas as pd
data = {'ID': [1,2], 'Valor': ['X', 'Y']}
df_guardar = pd.DataFrame(data)

# df_guardar.to_csv('salida.csv', index=False)
print("DataFrame que se guardaría en 'salida.csv':")
print(df_guardar.to_string(index=False)) # to_string para simular contenido

Contenido simulado de salida.csv (sin índice):

ID,Valor 1,X 2,Y

Escribir a Excel: df.to_excel()

Sintaxis: df.to_excel('nombre_archivo_salida.xlsx', sheet_name='MiHoja', index=False)

  • sheet_name: Nombre de la hoja donde se guardarán los datos.
  • Necesitarás openpyxl instalado.
# df_guardar.to_excel('salida_excel.xlsx', sheet_name='Resultados', index=False)

De manera similar, existen df.to_json(), df.to_sql(), etc.

Al escribir archivos, ten cuidado de no sobrescribir archivos importantes si no es tu intención.
